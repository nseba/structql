# structql

Go code generator for [hasura/go-graphql-client](https://github.com/hasura/go-graphql-client).

## Build & Test

```bash
go build ./...          # Build all
go test ./...           # Run all tests
go test -race ./...     # Run with race detector
go vet ./...            # Vet all
go test -run TestGolden -update  # Update golden files
```

## Project Structure

- `cmd/structql/` - CLI entry point
- `internal/config/` - YAML config loading and validation
- `internal/schema/` - GraphQL schema parsing (via gqlparser)
- `internal/query/` - Query file loading, parsing, validation
- `internal/codegen/` - Code generation: typemapper, structbuilder, renderer, generator
- `internal/output/` - File writing and obsolete file cleanup
- `testdata/` - Test schemas, queries, configs, golden files

## Key Conventions

- Generated files carry `// Code generated by structql. DO NOT EDIT.` header
- One file per GraphQL operation + one `types.go` for shared types
- Use `go/format.Source()` + `golang.org/x/tools/imports` for output formatting
- Golden file tests are the primary codegen correctness mechanism
- All public types/functions need doc comments

## Dependencies

- `github.com/vektah/gqlparser/v2` - GraphQL parsing
- `golang.org/x/tools` - goimports
- `gopkg.in/yaml.v3` - config parsing
